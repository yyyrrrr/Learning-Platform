# 前端框架使用指南

## 概述

本文档面向所有前端模块的负责人，说明框架提供了什么、没有提供什么，以及如何在框架基础上开发自己的模块。

**框架搭建人**：项目文档 + 全局协调负责人

## 三、如何开发自己的模块

### 3.1 找到你的页面文件

直接编辑 `src/views/<你的模块>/index.vue`，删除占位内容，写入真实业务逻辑。

**示例（图谱可视化 owner）**：

```
src/views/Graph/index.vue          ← 主页面，在这里开发
src/components/KnowledgeGraph/     ← 图谱渲染组件放这里
```

### 3.2 调用后端 API

每个模块对应一个 `src/api/modules/xxx.js`，里面已有函数骨架。

**使用方式**：

```js
// 在你的 .vue 文件中
import { getGraphData } from '@/api/modules/graph.js'

const data = await getGraphData()
```

- 响应已在拦截器中解包，直接拿 `data` 字段，不需要再写 `.data.data`
- 接口报错会自动弹出 `ElMessage.error`，不需要自己处理
- 如果后端接口路径和参数有变化，直接修改 `api/modules/xxx.js` 中对应的函数

**注意**：不要在 `.vue` 文件里直接使用 axios，必须通过 `api/modules/` 里的函数调用。

### 3.3 读取/修改全局状态

```js
import { useUserStore }     from '@/stores/user.js'
import { useKnowledgeStore } from '@/stores/knowledge.js'
import { useLearningStore }  from '@/stores/learning.js'

const userStore     = useUserStore()
const knowledgeStore = useKnowledgeStore()
const learningStore  = useLearningStore()

// 读取当前用户
userStore.userInfo

// 读取某知识点的学习进度
learningStore.getProgress(nodeId)

// 更新知识点进度（同时需要调用后端接口持久化）
learningStore.setProgress(nodeId, 'completed')
```

### 3.4 新增子页面/子路由

如果你的模块需要增加子路由（如习题详情页），**不要自行修改 `router/index.js`**，而是：

1. 在 `src/views/Exercise/` 下新建子页面文件
2. 向路由 owner 提交路由需求，说明：路径、组件位置、是否需要路由参数

### 3.5 新增公共组件

如果需要抽取公共组件，**不要直接往 `src/components/` 根目录塞**，而是：

1. 先在自己的模块目录下实现
2. 评估是否真正跨模块复用，若是，提 PR 并告知路由/全局协调 owner 评审

### 3.6 使用 ECharts

```js
import * as echarts from 'echarts'

// 在 onMounted 中初始化
const chartEl = ref(null)
onMounted(() => {
  const chart = echarts.init(chartEl.value)
  chart.setOption({ /* your option */ })
})
```

### 3.7 使用 D3.js

```js
import * as d3 from 'd3'

// 知识图谱可视化推荐使用 D3 force simulation
```

## 四、必须遵守的规范

### 4.1 文件改动边界

| 文件/目录 | 能改吗 | 说明 |
|----------|--------|------|
| `src/router/index.js` | ❌ 不能直接改 | 提需求给路由 owner |
| `src/api/index.js` | ❌ 不能改 | 拦截器由框架 owner 维护 |
| `src/stores/` 中已有字段 | ❌ 不能删/改语义 | 可以向 owner 提扩展需求 |
| `src/components/layout/` | ❌ 不能改 | 布局由框架 owner 维护 |
| `src/api/modules/你的模块.js` | ✅ 可以改 | 你的模块 API 文件 |
| `src/views/你的模块/` | ✅ 可以改 | 你的主目录 |
| `src/components/你的组件目录/` | ✅ 可以改 | 你的组件目录 |

### 4.2 API 调用规范

- 所有接口函数写在 `src/api/modules/xxx.js`，在 `.vue` 中只 import 函数，不直接用 axios
- 函数命名用动词开头：`getXxx`、`createXxx`、`updateXxx`、`deleteXxx`

### 4.3 路由 meta

每条路由都配置了 `meta.title`，浏览器标签会自动更新，不需要在页面里手动改 `document.title`。

## 五、本地启动

```bash
# 进入前端目录
cd frontend

# 安装依赖（已完成，clone 后首次运行需要）
npm install

# 启动开发服务器（需后端同时运行在 localhost:8080）
npm run dev

# 构建生产产物
npm run build
```

开发服务器地址：`http://localhost:5173`
