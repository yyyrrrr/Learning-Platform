# 待补充能力清单

## 说明

本文档与 `项目架构.md` 中的"待确定 / TODO 清单"并列。TODO 清单解决的是"用什么技术方案"的选型问题；本文档解决的是"现有架构设计中从未涉及"的能力缺口问题。

即使 TODO 清单全部合理确定，以下 5 项关键能力仍然需要额外补充设计和实现。

---

## 1. 算法执行步骤序列生成

**影响需求**：3.2 知识图谱交互式可视化、3.4.2 代码在线运行

**问题描述**：

当前架构里的"代码在线运行"（TODO 5.3）只能运行代码并得到输出结果，但无法生成**算法执行过程的步骤序列**（如"第 1 步指针移到索引 2，第 2 步交换元素 A 和 B"）。

参考项目 [algorithm-visualizer](https://github.com/algorithm-visualizer/algorithm-visualizer) 能实现动画回放，是因为它内置了 tracer 库，在代码执行时拦截并记录状态变化。本项目当前没有任何类似组件的设计。

**需要补充**：
- 执行步骤记录机制：代码插桩 / 解释器执行 / AST 分析，三选一
- 前后端步骤序列传输协议（数据结构定义）
- 前端 Visualizer 组件的"步骤序列 → 动画帧"解析逻辑

---

## 2. 学习路径推荐算法

**影响需求**：3.3.3 路径引导

**问题描述**：

Neo4j 可以存储"需要先学 A 再学 B"的依赖关系，但**拓扑排序 / 最短学习路径算法**后端没有实现位置。

需求要求"在目录中提示学习者的推荐学习顺序（如'需先学习二叉树定义，再学习二叉树遍历'）"，这不是简单的查询，而是需要图算法计算。

**需要补充**：
- 后端路径推荐服务（基于 Neo4j 依赖关系做拓扑排序或图遍历算法）
- 用户已学知识点过滤（排除已掌握节点）
- 推荐结果缓存（避免每次全图计算）

---

## 3. 知识漏洞识别算法

**影响需求**：3.5.2 统计分析

**问题描述**：

需求要求"结合知识图谱关联关系，识别学习者的知识漏洞（如某知识点未掌握导致相关知识点学习困难）"。这需要：
1. 查询某知识点的所有前驱依赖节点
2. 检查这些前驱节点的掌握状态
3. 向前传播标记"因前置知识缺失导致的潜在漏洞"

当前架构完全没有这一层算法设计。`learning_progress` 表只记录了单一知识点的状态，没有与知识图谱的依赖关系联动分析。

**需要补充**：
- 图遍历算法（从薄弱知识点向上回溯前驱依赖链）
- 掌握度传播规则（定义"未掌握的前置知识如何影响后置知识"的判断逻辑）
- 统计接口返回结构（需包含漏洞链路：如"二叉树遍历困难 → 前置：二叉树定义未掌握"）

---

## 4. 报告导出方案

**影响需求**：3.5.3 可视化统计报告

**问题描述**：

需求明确要求"支持报告导出，便于学习者回顾与总结"。但 TODO 清单和架构里都没有 PDF / Word 生成方案。

**需要补充**：
- 技术选型：
  - 后端方案：iText / Flying Saucer / OpenPDF（Java 生态）
  - 前端方案：html2canvas + jsPDF（纯前端截图生成）
- 报告模板设计（包含哪些统计图表、文字总结、学习建议）
- 导出接口设计（同步生成 vs 异步任务 + 下载链接）

---

## 5. 班级 / 分组数据模型

**影响需求**：3.5.2 统计分析（对比分析）

**问题描述**：

需求要求"可将学习者的学习数据与班级平均水平对比"。但当前 `user` 表结构中没有 `class_id`、`group_id`、`role_in_class` 等字段，无法做群体聚合统计。

**需要补充**：
- 扩展 `user` 表或新增 `class` / `user_class` 关联表
- 班级维度统计接口（平均分、完成率分布、薄弱知识点热力图）
- 数据权限控制（用户只能看到自己班级的对比数据）
