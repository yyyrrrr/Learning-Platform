# 《数据结构》智慧学习平台 - 项目架构说明

## 1. 技术栈

### 1.1 后端技术栈
- **Spring Boot 3.2.4** - 核心框架
- **MyBatis Plus 3.5.5** - ORM框架
- **Spring Data Neo4j** - Neo4j图数据库集成
- **MySQL 8.0** - 关系型数据库
- **Lombok** - 简化代码
- **Fastjson2** - JSON处理

### 1.2 前端技术栈（已确定）

根据 [要求.md] 和 [algorithm-visualizer](https://github.com/algorithm-visualizer/algorithm-visualizer) 确定

- **Vue 3** - 前端框架（来自 要求.md）
- **Element Plus** - UI 组件库（要求.md 中为 Element UI，因选用 Vue 3 故升级为 Element Plus）
- **ECharts / D3.js** - 知识图谱可视化（来自 要求.md）
- **Vue Router** - 前端路由（Vue 3 生态标配）
- **Axios** - HTTP 请求库（来自 [algorithm-visualizer](https://github.com/algorithm-visualizer/algorithm-visualizer)，见其 `package.json`）
- **Pinia** - 状态管理（Vue 3 官方推荐；AI 辅助生成代码时，Pinia 的简洁 API 和组合式风格一致性更好，误生成率远低于 Vuex）

## 3. 数据库设计

### 3.1 MySQL 表结构（12 张表）

1. **user** - 用户表
2. **knowledge_point** - 知识点表
3. **knowledge_content** - 知识点内容表
4. **code_example** - 代码示例表
5. **exercise** - 练习题表
6. **user_exercise** - 用户做题记录表
7. **learning_progress** - 学习进度表
8. **learning_record** - 学习行为记录表
9. **video** - 视频表
10. **video_watch_record** - 视频观看记录表
11. **ai_chat** - AI 对话记录表
12. **code_submission** - 代码提交记录表

### 3.2 Neo4j 图数据库
- 存储知识点节点和它们之间的关系（层级关系、关联关系）
- 支持知识图谱的可视化和遍历

**待确定：** MySQL `knowledge_point` 表与 Neo4j 节点之间的同步机制（双写 / 定时同步 / 人工导入）。

## 4. 核心功能架构

### 4.1 知识点详情页内容模板（已确定）

参考 [hello-algo](https://github.com/krahets/hello-algo) 的章节内容组织方式（定义 → 图示 → 代码 → 复杂度 → 小结）以及 [AlgoNote](https://github.com/itcharge/AlgoNote) 的知识点拆解逻辑，本平台知识点详情页采用以下模板：

```
知识点详情页
├── 1. 基本定义与核心性质
│   └── （可配合静态图示或简单动画）
├── 2. 操作原理与步骤讲解
├── 3. 代码实现（C++）
│   ├── 语法高亮、行号、一键复制（参考 hello-algo 代码展示体验）
│   └── 正常场景 + 异常场景覆盖
├── 4. 时间复杂度与空间复杂度分析
│   └── （参考 hello-algo 使用公式与对比表格呈现）
├── 5. 视频讲解（如有）
└── 6. 配套练习题
    └── 基础题 / 考研题 / 面试题（参考 AlgoNote 每章末尾固定练习题模式）
```

### 4.2 代码在线运行与可视化（已确定）

参考 [algorithm-visualizer](https://github.com/algorithm-visualizer/algorithm-visualizer) 的布局与交互模式：

- **页面布局**：左侧代码编辑器 + 右侧可视化/运行结果区域（参考 algorithm-visualizer 分栏布局）。
- **编辑器**：支持语法高亮、行号显示（可考虑 Ace Editor 或 Monaco Editor，参考 algorithm-visualizer 使用 react-ace）。
- **可视化组件**：按数据结构类型拆分独立渲染器（数组、树、图），接收后端返回的"步骤序列"并解析为动画（参考 algorithm-visualizer 的 tracer 机制与组件拆分策略）。
- **播放控制**：播放 / 暂停 / 步进 / 调速（参考 algorithm-visualizer 播放控制）。

**待确定：** 代码编译执行的后端环境（是否需要 Docker 沙箱 / OJ 判题系统）。

### 4.3 前端目录与路由结构（已确定）

参考 [hello-algo](https://github.com/krahets/hello-algo) 的 `chapter_XXX/index.md + 各小节` 层级结构，前端路由与知识图谱层级对齐：

```
/                          # 首页
/knowledge                 # 学习目录（基于知识图谱层级生成）
/knowledge/:module         # 模块首页（如 /knowledge/linear-list）
/knowledge/:module/:id     # 知识点详情页
/graph                     # 知识图谱交互式可视化
/playground                # 代码在线运行
/exercise                  # 习题模块
/statistics                # 学习跟踪与统计
/ai-chat                   # AI 辅助
```

## 8. 快速开始

1. 安装 MySQL 8.0，执行 `database-scripts/init.sql` 初始化数据库
2. 安装 Neo4j，配置连接信息
3. 修改 `application.yml` 中的数据库配置
4. 运行 `LearningPlatformApplication` 启动项目
5. 访问 `http://localhost:8080/api`
