# 习题模块说明文档

## 1. 功能概述

本模块负责提供数据结构与算法相关的练习题，支持多种题型，为学生提供在线练习、代码运行、自动判题和答案解析功能。

### 1.1 核心目标
- **题目管理**：提供数据结构与算法相关练习题库
- **在线练习**：支持多种题型的在线作答
- **代码运行**：编程题支持在线代码编辑、运行和调试
- **自动判题**：自动判题系统，即时反馈作答结果
- **学习跟踪**：记录做题历史，为学习统计提供数据
- **错题管理**：自动收集错题，支持错题重做

### 1.2 使用场景
| 场景 | 说明 |
|------|------|
| 随堂练习 | 学习知识点后，做相关练习题巩固 |
| 专项训练 | 针对特定知识点或算法进行集中训练 |
| 考前复习 | 通过做题检测知识掌握情况 |
| 面试准备 | 练习面试常见算法题 |

## 2. 题目类型与功能

### 2.1 题目类型

| 题型 | 功能支持 | 判题方式 |
|------|---------|---------|
| **单选题** | 单选选项、即时反馈 | 选项匹配 |
| **多选题** | 多选选项、部分得分 | 选项集合匹配 |
| **填空题** | 文本填空、关键字匹配 | 关键词匹配或正则表达式 |
| **编程题** | 代码编辑器、运行测试、提交判题 | 测试用例比对 |

### 2.2 题目属性

| 属性 | 说明 | 示例值 |
|------|------|--------|
| 难度 | 题目难度等级 | EASY/MEDIUM/HARD |
| 分类 | 题目用途分类 | BASIC/INTERVIEW/EXAM |
| 知识点 | 所属知识点 | 数组、链表、树、图等 |
| 通过率 | 历史通过率统计 | 85.5% |
| 时间限制 | 推荐答题时间（秒） | 30/60/120 |
| 初始代码 | 编程题初始代码模板 | 各语言模板代码 |

### 2.3 简化处理（大三可实现的范围）

**不需要做的**：
- ❌ 复杂的代码静态分析
- ❌ 多语言并发判题
- ❌ 代码抄袭检测
- ❌ 详细的代码性能分析报告

**需要做的**：
- ✅ 基于测试用例的判题（输入/输出匹配）
- ✅ 简单的编译错误信息提取
- ✅ 运行时间/内存限制检测
- ✅ 编程题代码高亮和基本补全

## 3. 数据库设计

### 3.1 已存在的表

#### 3.1.1 习题表（exercise）

已在项目架构中定义，结构如下：

```sql
CREATE TABLE exercise (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    knowledge_point_id BIGINT NOT NULL COMMENT '知识点ID',
    title VARCHAR(255) NOT NULL COMMENT '题目',
    description TEXT COMMENT '题目描述',
    type VARCHAR(50) NOT NULL COMMENT '题型(SINGLE_CHOICE, MULTIPLE_CHOICE, PROGRAMMING, FILL_BLANK)',
    difficulty VARCHAR(20) NOT NULL COMMENT '难度(EASY, MEDIUM, HARD)',
    category VARCHAR(50) DEFAULT 'BASIC' COMMENT '分类(BASIC, INTERVIEW, EXAM)',
    `options` TEXT COMMENT '选项(JSON格式)',
    answer TEXT COMMENT '正确答案',
    analysis TEXT COMMENT '题目解析',
    initial_code TEXT COMMENT '编程题初始代码',
    test_cases TEXT COMMENT '测试用例(JSON格式)',
    time_limit INT DEFAULT 60 COMMENT '时间限制(秒)',
    sort_order INT DEFAULT 0 COMMENT '排序字段',
    pass_rate DOUBLE DEFAULT 0 COMMENT '通过率',
    total_attempts INT DEFAULT 0 COMMENT '总尝试次数',
    correct_attempts INT DEFAULT 0 COMMENT '正确次数',
    deleted INT DEFAULT 0 COMMENT '逻辑删除标识',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='习题表';
```

**字段说明**：
- `options`：JSON格式存储题目选项，如 `["O(1)", "O(n)", "O(log n)", "O(n²)"]`
- `test_cases`：编程题测试用例，格式如 `[{"input": "1 2", "expectedOutput": "3"}]`
- `pass_rate`：动态计算的通过率 = `correct_attempts / total_attempts * 100`

**示例数据**：
```sql
-- 单选题
INSERT INTO exercise (knowledge_point_id, title, description, type, difficulty, options, answer, analysis)
VALUES (1, '数组的时间复杂度', '在数组中访问第i个元素的时间复杂度是：', 'SINGLE_CHOICE', 'EASY',
        '["O(1)", "O(n)", "O(log n)", "O(n²)"]', 'O(1)',
        '数组支持随机访问，通过索引直接定位元素，时间复杂度为O(1)');

-- 编程题
INSERT INTO exercise (knowledge_point_id, title, description, type, difficulty, initial_code, test_cases)
VALUES (2, '两数之和', '给定一个整数数组 nums 和一个整数目标值 target...', 'PROGRAMMING', 'EASY',
        '#include<bits/stdc++.h>\nusing namespace std;\nint main(){\n    // 在此编写代码\n    \n    return 0;\n}',
        '[{"input": "2 7 11 15\n9", "expectedOutput": "0 1"}, {"input": "3 2 4\n6", "expectedOutput": "1 2"}]');
```

   │ (知识体系)

## 6. 与其他模块的集成

### 6.1 与学习统计模块集成

**集成方式**：做题完成后自动记录学习行为。

```javascript
// 习题提交成功后
async function submitAnswer(answer) {
  const result = await exerciseApi.submit(answer);
  
  // 调用学习统计模块的记录接口
  await statisticsApi.recordLearning({
    action_type: 'COMPLETE_EXERCISE',
    target_id: exerciseId,
    action_data: { 
      correct: result.correct, 
      score: result.score,
      time_spent: timeSpent 
    },
    duration_seconds: timeSpent
  });
  
  return result;
}
```

## 7. 核心功能流程

### 7.1 选择题/填空题答题流程
```
开始答题
   ↓
显示题目（含描述、选项）
   ↓
用户作答
   ↓
提交答案
   ↓
后端验证答案 → 记录结果 → 更新通过率
   ↓
前端显示结果（正确/错误 + 解析）
```

### 7.2 编程题答题流程
```
开始编程题
   ↓
显示题目描述和测试用例
   ↓
用户在编辑器中编写代码
   ↓
[运行测试] → 执行测试用例 → 显示运行结果
   ↓
[提交判题] → 执行所有测试用例 → 记录结果
   ↓
显示判题结果（通过率、测试详情）
   ↓
更新用户做题记录和题目通过率
```

### 7.3 错题管理流程
```
用户答题错误
   ↓
自动记录到 user_exercise 表（is_correct = false）
   ↓
错题集页面读取错误记录
   ↓
支持按知识点筛选错题
   ↓
点击"重做"跳转到原题
   ↓
重新作答后更新记录
```

## 10. 注意事项

### 10.1 安全性考虑
- 编程题代码运行需在沙箱环境中
- 防止代码注入攻击
- 设置合理的运行时间和内存限制

### 10.2 性能考虑
- 习题列表分页加载，避免一次性加载过多
- 编程题测试用例分批执行
- 题目数据添加缓存

### 10.3 简化建议
- 如果时间紧张，可以先不做多语言支持（只支持C++）
- 判题系统可以先做简单的输入输出比对
- 错题导出功能可以后续迭代
