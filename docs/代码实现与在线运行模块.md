# 代码实现与在线运行模块

## 概述

本模块实现了基于 Piston 代码执行引擎的在线判题功能，支持代码自由运行和习题提交判题两种模式。

**模块负责人**：代码实现与在线运行负责人

---

## 一、快速启动（推荐）

项目根目录提供了一键启动脚本 `setup-and-run.js`，**首次配置或重新搭建环境时直接运行即可**，脚本会自动完成所有依赖安装和服务启动。

### 前置条件

运行脚本前，请确保以下工具已安装：

| 工具 | 说明 | 安装方式 |
|------|------|----------|
| Docker Desktop | 用于运行 Piston 代码执行引擎 | https://docs.docker.com/get-docker/ |
| Node.js (v18+) | 运行启动脚本 + 前端开发 | https://nodejs.org/ 或 `brew install node` |
| Maven (3.6+) | 构建和运行 Spring Boot 后端 | `brew install maven` 或 https://maven.apache.org/download.cgi |
| JDK 17+ | Java 运行环境 | `brew install openjdk@17` 或 https://adoptium.net/ |

> **注意**：安装 Docker Desktop 后请先启动它，确保 Docker 守护进程在运行。

### 运行脚本

```bash
# 在项目根目录执行
node setup-and-run.js
```

脚本会依次自动完成：

1. 检查所有前置工具是否已安装
2. 启动 Piston Docker 容器（代码执行引擎）
3. 安装 C++、C、Python 语言运行时（已安装的自动跳过）
4. 安装前端 npm 依赖（已有 node_modules 则跳过）
5. 编译后端 Java 代码
6. 同时启动 Spring Boot 后端 + Vite 前端开发服务器

启动完成后打开浏览器访问：

- **习题测试页面**：http://localhost:5173/exercise/1
- 前端开发服务器：http://localhost:5173
- 后端 API：http://localhost:8080/api
- Piston 引擎：http://localhost:2000

按 `Ctrl+C` 可同时停止所有服务。

## 二、手动启动

如果需要单独启动某个服务，或脚本运行失败，可按以下步骤手动操作。

### 2.1 启动 Piston 代码执行引擎

Piston 是一个自托管的代码沙箱，通过 Docker 运行。配置文件为项目根目录的 `docker-compose.yml`。

```bash
# 在项目根目录执行
docker compose up -d
```

验证 Piston 已成功启动：

```bash
curl http://localhost:2000/api/v2/runtimes
# 应返回 JSON 数组（首次安装语言包前可能为空数组 []）
```

### 2.2 安装语言运行时（首次启动后执行一次）

Piston 容器启动后，需要单独安装各语言的编译器/解释器：

```bash
# 安装 C++（版本 10.2.0）
curl -X POST http://localhost:2000/api/v2/packages \
  -H 'Content-Type: application/json' \
  -d '{"language":"c++","version":"10.2.0"}'

# 安装 C（版本 10.2.0）
curl -X POST http://localhost:2000/api/v2/packages \
  -H 'Content-Type: application/json' \
  -d '{"language":"c","version":"10.2.0"}'

# 安装 Python（版本 3.10.0）
curl -X POST http://localhost:2000/api/v2/packages \
  -H 'Content-Type: application/json' \
  -d '{"language":"python","version":"3.10.0"}'
```

> 安装过程需要联网下载，每个语言包约几十 MB，请耐心等待。**语言包数据持久化在 Docker Volume 中，重启容器不需要重新安装。**

验证安装成功：

```bash
curl http://localhost:2000/api/v2/runtimes
# 应能看到 c++、c、python 的条目
```

### 2.3 启动后端

需要先确保 MySQL 和 Neo4j 已启动，详见 `docs/项目架构.md`。

```bash
# 在项目根目录执行
mvn spring-boot:run
```

后端启动成功后控制台会打印 `Started LearningPlatformApplication`，监听端口 `8080`。

### 2.4 启动前端

```bash
cd frontend
npm install      # 首次运行需要安装依赖（约 30MB，monaco-editor 较大）
npm run dev
```

前端启动后访问 http://localhost:5173。

## 三、已完成的功能

### 3.1 代码自由运行（Run）

用户在编辑器中编写代码，选择当前测试用例作为输入，点击「运行」后获得标准输出、标准错误和编译错误信息。

### 3.2 习题提交判题（Submit）

用户提交代码后，后端对所有测试用例逐一执行，返回每个用例的通过/失败状态、实际输出与期望输出对比，最终给出通过数/总数和判题结论。

判题状态说明：

| status | 含义 |
|--------|------|
| `ACCEPTED` | 全部用例通过 |
| `WRONG_ANSWER` | 存在用例答案错误 |
| `COMPILE_ERROR` | 编译失败 |
| `RUNTIME_ERROR` | 运行时异常（含超时） |

### 3.3 支持的编程语言

| 语言 | 选项值 | Monaco 语言标识 |
|------|--------|-----------------|
| C++ | `c++` | `cpp` |
| C | `c` | `c` |
| Python | `python` | `python` |

切换语言时编辑器自动填入对应代码模板。

## 四、技术架构

### 4.1 执行引擎

使用 **Piston**（自托管 Docker 容器）作为代码沙箱。

- 配置文件：`docker-compose.yml`（项目根目录）
- 服务地址：`http://localhost:2000`
- 需要 `privileged: true` 权限以支持 isolate 沙箱隔离

**stdin 注入机制**：测试用例的 `input` 字段通过 stdin pipe 注入给程序进程。程序用 `cin`/`scanf` 正常读取即可，与在本地终端运行完全相同。

### 4.2 后端代码结构

```
src/main/java/com/learningplatform/
├── controller/CodeController.java       ← REST 接口入口（/api/code/run、/api/code/submit）
├── service/CodeExecutionService.java    ← 业务逻辑，调用 Piston API
└── dto/
    ├── CodeRunRequest.java              ← 运行请求 DTO
    ├── CodeRunResponse.java             ← 运行结果 DTO
    ├── CodeSubmitRequest.java           ← 提交请求 DTO
    ├── CodeSubmitResponse.java          ← 提交结果 DTO
    ├── TestCase.java                    ← 测试用例 DTO
    └── TestResult.java                  ← 单个用例判题结果 DTO
```

### 4.3 前端代码结构

```
frontend/src/
├── components/CodeEditor/
│   └── CodeEditor.vue                  ← Monaco 编辑器封装（v-model 双向绑定）
├── views/Exercise/
│   ├── index.vue                       ← 习题列表页（占位，待完善）
│   └── ExerciseDetail.vue              ← 习题详情 + 编辑器主页面
└── api/modules/code.js                 ← runCode / submitCode API 封装
```

## 五、API 接口

**Base URL**: `http://localhost:8080/api`

> 前端开发模式下 `/api` 请求自动代理到后端，无需配置跨域。

### 5.1 代码运行

**POST** `/code/run`

请求体：

```json
{
  "language": "c++",
  "code": "#include<bits/stdc++.h>\nusing namespace std;\nint main(){\n    int a,b;\n    cin>>a>>b;\n    cout<<a+b<<endl;\n    return 0;\n}",
  "stdin": "3 5"
}
```

响应体：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "stdout": "8\n",
    "stderr": "",
    "compileError": null
  }
}
```

### 5.2 代码提交判题

**POST** `/code/submit`

请求体：

```json
{
  "language": "c++",
  "code": "...",
  "testCases": [
    { "input": "3 5", "expectedOutput": "8" },
    { "input": "0 0", "expectedOutput": "0" }
  ]
}
```

响应体：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "status": "ACCEPTED",
    "passCount": 2,
    "totalCount": 2,
    "errorMessage": null,
    "testResults": [
      { "passed": true, "input": "3 5", "expectedOutput": "8", "actualOutput": "8" },
      { "passed": true, "input": "0 0", "expectedOutput": "0", "actualOutput": "0" }
    ]
  }
}
```

## 六、前端组件使用

### 6.1 在其他页面集成 ExerciseDetail

路径：`src/views/Exercise/ExerciseDetail.vue`

通过 Props 传入题目数据：

```vue
<ExerciseDetail :exercise="exerciseData" />
```

```js
const exerciseData = {
  title: '两数之和',
  difficulty: 'EASY',           // 'EASY' | 'MEDIUM' | 'HARD'
  description: '题目描述文本',
  testCases: [
    { input: '3 5', expectedOutput: '8' },
    { input: '0 0', expectedOutput: '0' }
  ]
}
```

路由地址：`/exercise/:id`

### 6.2 调用代码执行 API

```js
import { runCode, submitCode } from '@/api/modules/code.js'

// 运行代码（返回值已由 axios 拦截器解包，直接是业务数据）
const result = await runCode({
  language: 'c++',
  code: '...',
  stdin: '3 5'
})
console.log(result.stdout)        // '8\n'
console.log(result.compileError)  // null 表示编译成功

// 提交判题
const result = await submitCode({
  language: 'c++',
  code: '...',
  testCases: [{ input: '3 5', expectedOutput: '8' }]
})
console.log(result.status)        // 'ACCEPTED'
console.log(result.passCount)     // 1
```
