# 代码实现与在线运行模块

## 概述

本模块实现了基于 Piston 代码执行引擎的在线判题功能，支持代码自由运行和习题提交判题两种模式。

**模块负责人**：代码实现与在线运行负责人

---

## 一、已完成的功能

### 1.1 代码自由运行（Run）

用户可以在编辑器中编写代码，选择自定义输入，点击「运行」后获得标准输出、标准错误和编译错误信息。

### 1.2 习题提交判题（Submit）

用户提交代码后，后端对所有测试用例逐一执行，返回每个用例的通过/失败状态、实际输出与期望输出对比，最终给出通过数/总数和判题结论（ACCEPTED / WRONG_ANSWER / COMPILE_ERROR / RUNTIME_ERROR）。

### 1.3 支持的编程语言

| 语言   | 选项值   | Monaco 语言标识 |
| ------ | -------- | --------------- |
| C++    | `c++`    | `cpp`           |
| C      | `c`      | `c`             |
| Python | `python` | `python`        |

### 1.4 代码模板

切换语言时编辑器自动填入对应模板：

```cpp
// C++
#include<bits/stdc++.h>
using namespace std;
int main(){
    // 在此编写代码
    return 0;
}
```

```c
// C
#include<stdio.h>
int main(){
    // 在此编写代码
    return 0;
}
```

```python
# Python
# 在此编写代码
```

## 二、技术架构

### 2.1 执行引擎

使用 **Piston**（自托管 Docker 容器）作为代码沙箱执行引擎。

- Docker Compose 配置路径：`docker/docker-compose.yml`
- Piston 服务地址（内部）：`http://localhost:2000`
- 需要 `privileged: true` 权限以支持 isolate 沙箱

启动方式：

```bash
cd docker
docker compose up -d
```

首次启动后需要安装语言运行时（只需执行一次）：

```bash
# 安装 C/C++ 和 Python
curl -s http://localhost:2000/api/v2/packages | python3 -m json.tool | grep '"language"'
curl -X POST http://localhost:2000/api/v2/packages \
  -H 'Content-Type: application/json' \
  -d '{"language":"c++","version":"10.2.0"}'
curl -X POST http://localhost:2000/api/v2/packages \
  -H 'Content-Type: application/json' \
  -d '{"language":"python","version":"3.10.0"}'
```

## 三、API 接口

**Base URL**: `http://localhost:8080/api`

### 3.1 代码运行

**POST** `/code/run`

请求体：

```json
{
  "language": "c++",
  "code": "#include<bits/stdc++.h>\n...",
  "stdin": "3 5"
}
```

响应体：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "stdout": "8\n",
    "stderr": "",
    "compileError": null
  }
}
```

字段说明：

| 字段           | 类型   | 说明                             |
| -------------- | ------ | -------------------------------- |
| `stdout`       | String | 标准输出                         |
| `stderr`       | String | 运行时错误输出                   |
| `compileError` | String | 编译错误信息，非空时表示编译失败 |

### 3.2 代码提交判题

**POST** `/code/submit`

请求体：

```json
{
  "language": "c++",
  "code": "#include<bits/stdc++.h>\n...",
  "testCases": [
    { "input": "3 5", "expectedOutput": "8" },
    { "input": "0 0", "expectedOutput": "0" }
  ]
}
```

响应体：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "status": "ACCEPTED",
    "passCount": 2,
    "totalCount": 2,
    "errorMessage": null,
    "testResults": [
      {
        "passed": true,
        "input": "3 5",
        "expectedOutput": "8",
        "actualOutput": "8"
      },
      {
        "passed": true,
        "input": "0 0",
        "expectedOutput": "0",
        "actualOutput": "0"
      }
    ]
  }
}
```

判题状态说明：

| status          | 含义             |
| --------------- | ---------------- |
| `ACCEPTED`      | 全部用例通过     |
| `WRONG_ANSWER`  | 存在用例答案错误 |
| `COMPILE_ERROR` | 编译失败         |
| `RUNTIME_ERROR` | 运行时异常       |

## 四、前端组件使用

### 4.1 CodeEditor 组件

路径：`src/components/CodeEditor/CodeEditor.vue`

基于 `monaco-editor-vue3` 封装，支持 `v-model` 双向绑定。

```vue
<CodeEditor v-model="code" :language="'cpp'" />
```

Props：

| 属性         | 类型   | 默认值  | 说明                     |
| ------------ | ------ | ------- | ------------------------ |
| `modelValue` | String | `''`    | 编辑器内容（v-model）    |
| `language`   | String | `'cpp'` | 语言标识（cpp/c/python） |

### 4.2 ExerciseDetail 组件

路径：`src/views/Exercise/ExerciseDetail.vue`

LeetCode 风格的习题详情页，左右分栏布局（题目描述 + 代码编辑器）。

通过 Props 接收题目数据：

```js
// Props 结构
{
  title: String,          // 题目标题
  difficulty: String,     // 'EASY' | 'MEDIUM' | 'HARD'
  description: String,    // 题目描述
  codeTemplate: String,   // 初始代码模板
  testCases: [
    { input: String, expectedOutput: String }
  ]
}
```

路由地址：`/exercise/:id`

### 4.3 调用 API

```js
import { runCode, submitCode } from "@/api/modules/code.js";

// 运行代码
const result = await runCode({
  language: "c++",
  code: "...",
  stdin: "3 5",
});
// result 直接是 data 字段（拦截器已解包）
console.log(result.stdout);

// 提交判题
const result = await submitCode({
  language: "c++",
  code: "...",
  testCases: [{ input: "3 5", expectedOutput: "8" }],
});
console.log(result.status); // 'ACCEPTED'
```

## 五、本地启动

### 5.1 启动 Piston 代码执行引擎

```bash
# 进入 docker 目录（根据实际路径调整）
docker compose up -d
```

确认 Piston 启动成功：

```bash
curl http://localhost:2000/api/v2/runtimes
```

### 5.2 启动后端

```bash
# 项目根目录
mvn spring-boot:run
```

### 5.3 启动前端

```bash
cd frontend
npm install
npm run dev
```

访问 `http://localhost:5173/exercise/1` 即可进入习题详情页。

## 六、注意事项

- Piston 容器需要 `privileged: true`，请确保 Docker 有足够权限。
- `monaco-editor` 体积较大，首次 `npm install` 会多下载约 30MB 依赖。
- 代码执行有超时限制（Piston 默认 3 秒），超时会返回 `RUNTIME_ERROR`。
- 前端 `/api` 请求在开发模式下自动代理到 `http://localhost:8080`，无需额外配置跨域。
