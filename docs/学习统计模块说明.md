# 学习跟踪与统计分析模块说明

## 1. 功能概述

本模块负责跟踪记录学生的学习行为数据，并基于这些数据进行多维度统计分析，以可视化图表的形式呈现给学习者，帮助学生了解自己的学习进度、掌握情况和薄弱环节。

### 1.1 核心目标
- **学习跟踪**：自动记录学生的学习行为（浏览知识点、做题、看视频等）
- **进度统计**：展示整体和各模块的学习进度
- **掌握度分析**：基于做题正确率评估知识点掌握程度
- **可视化报告**：用图表直观展示学习数据
- **学习建议**：根据分析结果给出简单的复习建议

### 1.2 使用场景
| 场景 | 说明 |
|------|------|
| 学生查看个人主页 | 展示整体学习概况（进度、时长、做题数） |
| 学完一个模块后 | 查看该模块的掌握情况和薄弱点 |
| 复习阶段 | 查看错题集和推荐复习的知识点 |
| 阶段总结 | 导出学习报告，回顾学习历程 |

## 2. 需要采集的数据

### 2.1 学习行为数据

**不需要埋点追踪所有操作**，只记录关键行为：

| 行为类型 | 触发时机 | 记录内容 |
|---------|---------|---------|
| **浏览知识点** | 进入知识点详情页并停留>30 秒 | 知识点 ID、学习时长、时间戳 |
| **完成练习题** | 提交练习题并判分后 | 题目 ID、是否正确、得分、耗时、错题答案 |
| **观看视频** | 关闭视频或章节结束时 | 视频 ID、观看进度%、观看时长 |
| **代码练习** | 在代码运行页面提交代码 | 知识点 ID、运行次数、是否成功 |

### 2.2 数据来源

| 数据 | 来源模块 | 如何获取 |
|-----|---------|---------|
| 知识点浏览记录 | 知识点详情页 | 前端调用记录接口 |
| 练习题结果 | 习题模块 | 做题完成后推送数据 |
| 视频观看记录 | 视频模块 | 视频播放结束时调用 |
| 代码提交记录 | 代码运行模块 | 代码提交时推送数据 |

### 2.3 简化处理（大三可实现的范围）

**不需要做的**：
- ❌ 实时追踪鼠标移动、页面滚动等细粒度行为
- ❌ 复杂的学习习惯预测算法
- ❌ 与班级平均水平的对比分析（需要班级数据模型，当前数据库不支持）

**需要做的**：
- ✅ 简单的学习时长累计（前端计时，后端累加）
- ✅ 做题正确率统计（直接查询做题记录表计算）
- ✅ 知识点掌握度标记（根据正确率分档：>80% 掌握，50-80% 基本掌握，<50% 未掌握）

## 3. 数据库设计

### 3.1 需要新增的表

#### 3.1.1 学习行为记录表（learning_record）

记录学生的关键学习行为。

```sql
CREATE TABLE learning_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户 ID',
    action_type VARCHAR(50) NOT NULL COMMENT '行为类型：VIEW_KNOWLEDGE/COMPLETE_EXERCISE/WATCH_VIDEO/CODE_PRACTICE',
    target_id BIGINT NOT NULL COMMENT '目标对象 ID（知识点 ID/ 题目 ID/ 视频 ID）',
    action_data JSON COMMENT '行为附加数据（如正确率、观看进度等）',
    duration_seconds INT COMMENT '持续时长（秒）',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '行为发生时间',
    INDEX idx_user_time (user_id, create_time),
    INDEX idx_user_action (user_id, action_type)
);
```

**字段说明**：
- `action_type`：区分不同类型的学习行为
- `target_id`：对应类型的对象 ID（知识点/题目/视频）
- `action_data`：JSON 格式存储额外信息，如 `{"correct": true, "score": 90}`
- `duration_seconds`：学习时长（浏览知识点时长、视频观看时长等）

**示例数据**：
```sql
-- 浏览知识点 5 分钟
INSERT INTO learning_record (user_id, action_type, target_id, action_data, duration_seconds)
VALUES (1, 'VIEW_KNOWLEDGE', 15, '{"module": "TREE"}', 300);

-- 完成练习题（正确）
INSERT INTO learning_record (user_id, action_type, target_id, action_data, duration_seconds)
VALUES (1, 'COMPLETE_EXERCISE', 88, '{"correct": true, "score": 100}', 120);

-- 观看视频 80%
INSERT INTO learning_record (user_id, action_type, target_id, action_data, duration_seconds)
VALUES (1, 'WATCH_VIDEO', 5, '{"progress": 80}', 480);
```

#### 3.1.2 学习进度表（learning_progress）

记录学生对每个知识点的掌握状态。

```sql
CREATE TABLE learning_progress (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户 ID',
    knowledge_point_id BIGINT NOT NULL COMMENT '知识点 ID',
    status VARCHAR(20) DEFAULT 'NOT_STARTED' COMMENT '状态：NOT_STARTED/LEARNING/COMPLETED',
    mastery_level VARCHAR(20) DEFAULT 'UNKNOWN' COMMENT '掌握程度：UNKNOWN/MASTERED/BASIC/NOT_MASTERED',
    total_duration INT DEFAULT 0 COMMENT '累计学习时长（秒）',
    exercise_count INT DEFAULT 0 COMMENT '做题数量',
    correct_count INT DEFAULT 0 COMMENT '正确数量',
    last_learned_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '最后学习时间',
    UNIQUE KEY uk_user_kp (user_id, knowledge_point_id),
    INDEX idx_user_status (user_id, status)
);
```

**字段说明**：
- `status`：学习状态（未开始/学习中/已完成）
- `mastery_level`：掌握程度（未知/掌握/基本掌握/未掌握）
- `total_duration`：累计学习时长（秒）
- `exercise_count` / `correct_count`：做题统计，用于计算正确率

**掌握度计算规则**（简化版）：
```
正确率 = correct_count / exercise_count（如果 exercise_count > 0）

IF 正确率 >= 0.8 THEN mastery_level = 'MASTERED'
ELSE IF 正确率 >= 0.5 THEN mastery_level = 'BASIC'
ELSE IF 正确率 > 0 THEN mastery_level = 'NOT_MASTERED'
ELSE mastery_level = 'UNKNOWN'

IF exercise_count > 0 AND 正确率 >= 0.5 THEN status = 'COMPLETED'
ELSE IF total_duration > 60 THEN status = 'LEARNING'
ELSE status = 'NOT_STARTED'
```

## 4. 统计维度与展示方式

### 4.1 学习概况（首页/统计页顶部）

**展示内容**：
- 总学习时长（小时）
- 已学知识点数 / 总知识点数
- 已完成题目数 / 总题目数
- 平均正确率

**图表形式**：4 个数字卡片（不需要图表）

**数据来源**：
```sql
-- 总学习时长
SELECT SUM(duration_seconds) / 3600 FROM learning_record WHERE user_id = ?;

-- 已学知识点数
SELECT COUNT(DISTINCT knowledge_point_id) FROM learning_progress 
WHERE user_id = ? AND status != 'NOT_STARTED';

-- 做题统计
SELECT COUNT(*) as total, SUM(CASE WHEN correct = 1 THEN 1 ELSE 0 END) as correct
FROM user_exercise WHERE user_id = ?;
```

### 4.2 学习进度分析（柱状图）

**展示内容**：各模块（集合、线性表、树、图、查找、排序）的学习进度

**图表形式**：分组柱状图
- X 轴：6 个知识模块
- Y 轴：进度百分比（0-100%）
- 每个模块显示两根柱子：知识点完成率、题目完成率

**数据来源**：
```sql
SELECT kp.module, 
       COUNT(DISTINCT kp.id) as total_kp,
       COUNT(DISTINCT lp.knowledge_point_id) as learned_kp
FROM knowledge_point kp
LEFT JOIN learning_progress lp ON kp.id = lp.knowledge_point_id AND lp.user_id = ?
GROUP BY kp.module;
```

### 4.3 知识点掌握情况（雷达图）

**展示内容**：各模块的掌握度评分

**图表形式**：雷达图
- 6 个维度：6 个知识模块
- 每个维度评分 = 该模块下所有知识点的平均掌握率

**掌握率计算**：
```
模块掌握率 = Σ(知识点掌握度分数) / 知识点数量

掌握度分数：MASTERED=100, BASIC=60, NOT_MASTERED=20, UNKNOWN=0
```

### 4.4 学习趋势（折线图）

**展示内容**：近 7 天/近 30 天的学习时长变化

**图表形式**：折线图
- X 轴：日期（最近 7 天或 30 天）
- Y 轴：学习时长（小时）

**数据来源**：
```sql
SELECT DATE(create_time) as date, SUM(duration_seconds) / 3600 as hours
FROM learning_record
WHERE user_id = ? AND create_time >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY DATE(create_time)
ORDER BY date;
```

### 4.5 薄弱知识点（列表）

**展示内容**：掌握度较低的知识点列表（最多显示 10 个）

**展示形式**：表格
| 知识点名称 | 所属模块 | 掌握程度 | 做题正确率 | 建议 |
|-----------|---------|---------|-----------|------|
| 二叉树遍历 | 树 | 未掌握 | 40% | 建议复习 |
| 快速排序 | 排序 | 基本掌握 | 65% | 继续练习 |

**数据来源**：
```sql
SELECT kp.name, kp.module, lp.mastery_level, 
       lp.correct_count * 100 / lp.exercise_count as correct_rate
FROM learning_progress lp
JOIN knowledge_point kp ON lp.knowledge_point_id = kp.id
WHERE lp.user_id = ? AND lp.mastery_level IN ('NOT_MASTERED', 'BASIC')
ORDER BY correct_rate ASC
LIMIT 10;
```

### 4.6 错题集（列表 + 导出）

**展示内容**：做错的题目列表

**展示形式**：可筛选的列表
- 按模块筛选
- 按时间筛选
- 支持重新练习（跳转题目）

**数据来源**：
```sql
SELECT ue.exercise_id, e.title, e.module, ue.correct, ue.submit_time
FROM user_exercise ue
JOIN exercise e ON ue.exercise_id = e.id
WHERE ue.user_id = ? AND ue.correct = 0
ORDER BY ue.submit_time DESC;
```

## 9. 注意事项

### 9.1 性能考虑
- 统计数据查询可能较慢，建议添加缓存（如 Redis 或简单的内存缓存）
- 学习行为记录是高频操作，避免在记录时做复杂计算

### 9.2 数据一致性
- `learning_progress` 表的掌握度需要根据做题记录定期更新
- 可以每次做题后实时更新，或者每天定时任务批量更新

### 9.3 简化建议
- 如果时间紧张，可以不做"学习趋势"和"报告导出"功能
- 优先保证概况、进度、掌握度这三个核心统计
- 错题集可以先做简单列表，后续再优化筛选功能
