# 数据层协同规范

## 1. 工作流程

### 阶段1: 知识点梳理
1. 各模块负责人在 `data-layer/data/dictionary/knowledge_point_dict.md` 中填写知识点清单
2. 数据层负责人审核并统一编号
3. 确定知识点层级关系和关联关系

### 阶段2: 数据录入
1. 各模块负责人在 `data-layer/scripts/mysql/mysql-modules/` 目录下编写对应模块的SQL
2. 数据层负责人合并并执行到MySQL
3. 生成Neo4j导入CSV文件
4. 执行Neo4j导入脚本

### 阶段3: 数据维护
1. 任何知识点变更需通过PR提交
2. 数据层负责人审核并执行同步
3. 定期校验MySQL与Neo4j一致性

## 2. 提交规范

### 2.1 SQL文件命名
- 模块数据: `data-layer/scripts/mysql/mysql-modules/XX_模块名.sql`
- 格式:
```sql
-- 模块: 线性表
-- 负责人: XXX
-- 日期: 2026-XX-XX

INSERT INTO knowledge_point (neo4j_id, name, code, parent_id, module, level, difficulty, description, core_points)
VALUES 
('KP_LL_001', '数组', 'ARRAY', NULL, 'LINEAR_LIST', 3, 'EASY', '数组是一种线性数据结构...', '内存连续、随机访问O(1)'),
('KP_LL_002', '链表', 'LINKED_LIST', NULL, 'LINEAR_LIST', 3, 'MEDIUM', '链表是一种...','指针操作、动态分配');
```

### 2.2 CSV文件格式

**节点文件**: `data/neo4j_import/nodes/knowledge_points.csv`
```csv
neo4j_id,name,code,module,level,difficulty,description,core_points,mysql_id
KP_LL_001,数组,ARRAY,LINEAR_LIST,3,EASY,"数组是一种线性数据结构...","内存连续、随机访问O(1)",1
KP_LL_002,链表,LINKED_LIST,LINEAR_LIST,3,MEDIUM,"链表是一种...","指针操作、动态分配",2
```

**层级关系文件**: `data/neo4j_import/relationships/parent_of.csv`
```csv
:START_ID,:END_ID,order
KP_TREE_001,KP_TREE_002,1
KP_TREE_001,KP_TREE_003,2
```

**关联关系文件**: `data/neo4j_import/relationships/related_to.csv`
```csv
:START_ID,:END_ID,type,strength
KP_LL_001,KP_LL_002,PREREQUISITE,5
KP_TREE_002,KP_TREE_005,EXTENSION,3
```

## 3. 协作原则

### 3.1 数据层负责人职责
- 维护数据字典和命名规范
- 审核所有数据变更PR
- 执行MySQL和Neo4j数据同步
- 定期校验数据一致性

### 3.2 模块负责人职责
- 填写负责模块的知识点清单
- 编写对应模块的SQL插入语句
- 描述知识点层级和关联关系
- 不直接修改其他模块数据

### 3.3 变更流程
- **新增知识点**: 提交PR → 数据层审核 → 执行同步
- **修改知识点**: 提交PR → 数据层审核 → 执行同步
- **删除知识点**: 提交PR → 数据层审核 → 执行同步(逻辑删除)

## 4. Docker环境配置

### 4.1 独立数据层Docker

数据层使用独立的 `data-layer/` 目录，与后端完全分离：

```
Learning-Platform/
├── data-layer/                     # 数据层统一目录
│   ├── docker/                     # Docker配置
│   │   ├── docker-compose.yml      # MySQL + Neo4j
│   │   ├── .env                    # 环境变量
│   │   └── neo4j-entrypoint.sh     # 自动导入入口
│   ├── scripts/                    # 数据库脚本
│   │   ├── mysql/                  # MySQL脚本
│   │   │   ├── init.sql            # 初始化脚本
│   │   │   └── mysql-modules/      # 各模块数据
│   │   └── neo4j/                  # Neo4j脚本
│   │       ├── schema.cql          # 索引和约束
│   │       └── auto_import.sh      # 自动导入脚本
│   └── data/                       # 数据文件
│       ├── neo4j_import/           # Neo4j导入数据
│       └── dictionary/             # 数据字典
└── ... (其他项目文件)
```

**优势**:
- 数据层自主管理，不依赖后端
- 后端可以随时连接/断开，不影响数据层
- 符合关注点分离原则

### 4.2 启动服务
```bash
# 进入数据层Docker目录
cd data-layer/docker

# 启动MySQL和Neo4j
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 4.3 访问地址
- MySQL: `localhost:13306` (root/root123)
- Neo4j: `http://localhost:17474` (neo4j/neo4j123)
- Neo4j Bolt: `bolt://localhost:17687`

**注意**: 端口已调整为 13306/17474/17687，避免与本地MySQL/Neo4j服务冲突。

### 4.4 数据持久化
- MySQL数据: `mysql_data` volume
- Neo4j数据: `neo4j_data` volume
- 导入文件: `../data/neo4j_import` 目录

### 4.6 Neo4j数据一键导入

Neo4j数据导入支持**Docker自动导入**：

#### Docker自动导入（唯一方式）

启动Docker时会自动导入数据：
```bash
cd data-layer/docker
docker-compose up -d
```

首次启动时会自动执行导入，后续启动会检测已有数据并跳过。

#### 手动重新导入

如需重新导入数据：
```bash
cd data-layer/docker
# 停止并删除数据卷（⚠️ 会清空所有数据）
docker-compose down -v
# 重新启动
docker-compose up -d
```

#### 导入流程

1. 创建索引和约束
2. 导入知识点节点
3. 导入层级关系
4. 导入关联关系
5. 验证导入结果

**CSV文件位置**: `../data/neo4j_import/` 目录

### 4.7 示例文件使用说明

`data-layer/scripts/mysql/mysql-modules/` 目录下的所有 `.sql` 文件都是**示例文件**，用于展示数据格式和编写规范，**不是真实数据**。

**如何使用示例文件**:

1. **参考格式**: 示例文件展示了正确的SQL编写格式
   ```sql
   -- 模块: 线性表
   -- 负责人: XXX
   -- 日期: 2024-XX-XX
   
   INSERT INTO knowledge_point (...)
   VALUES (...);
   ```

2. **编写自己的数据**: 
   - 复制示例文件的格式
   - 替换为你的模块真实数据
   - 保留注释头（模块、负责人、日期）

3. **提交规范**:
   - 在 `data/dictionary/knowledge_point_dict.md` 填写知识点清单
   - 在 `data-layer/scripts/mysql/mysql-modules/` 编写对应模块的SQL
   - 通过PR提交，数据层负责人审核

4. **注意事项**:
   - 不要直接修改示例文件
   - 示例文件仅用于参考格式
   - 真实数据需要通过数据层负责人审核后导入

### 4.5 与后端协作

**后端连接数据库只需配置**:
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:13306/learning_platform
    username: root
    password: root123
  neo4j:
    uri: bolt://localhost:17687
    authentication:
      username: neo4j
      password: neo4j123
```

**数据层负责维护数据库运行，后端负责连接使用，互不干扰。**

## 5. 数据同步流程

### 5.1 MySQL → Neo4j 导入步骤
1. 确认MySQL数据已更新
2. 从MySQL导出知识点数据到CSV
3. 准备关系数据CSV
4. 执行Neo4j导入脚本
5. 校验导入结果

### 5.2 一致性校验
```cypher
// 检查MySQL和Neo4j知识点数量是否一致
MATCH (kp:KnowledgePoint)
RETURN count(kp) as neo4j_count

// 检查是否有孤立的知识点节点
MATCH (kp:KnowledgePoint)
WHERE NOT (kp)-[:PARENT_OF|RELATED_TO]-()
RETURN kp.neo4j_id, kp.name
```

## 6. 注意事项

1. **编号规则**: 严格按照 `KP_模块前缀_序号` 格式
2. **数据备份**: 每次大批量导入前备份MySQL和Neo4j
3. **冲突处理**: 多人同时提交时，数据层负责人协调合并
4. **版本管理**: 重要数据变更在PR描述中标注版本号
5. **测试环境**: 先在测试环境验证，确认无误后再同步到正式环境
