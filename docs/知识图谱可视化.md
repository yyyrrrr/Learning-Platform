# 知识图谱交互式可视化模块说明

## 1. 模块概述

本模块负责知识图谱的前端可视化展示，采用 ECharts 力导向图实现，支持缩放、拖拽、搜索、高亮、节点点击查看详情等交互功能。

**技术方案**：Vue 3 + ECharts 6.0（graph 类型，力导向布局）

---

## 2. 数据层

### 2.1 Neo4j 数据文件

**文件**：`data-layer/data/neo4j_import/knowledge_points.cypher`

补充了覆盖 6 大模块的完整知识图谱数据：

| 模块 | 节点数 | 根节点 |
|------|--------|--------|
| 集合 (SET) | 2 | 集合 |
| 线性表 (LINEAR_LIST) | 9 | 线性表 → 栈、队列、串、数组、广义表 |
| 树 (TREE) | 8 | 树 → 二叉树、二叉排序树、B树 |
| 图 (GRAPH) | 10 | 图 → 存储结构、DFS、BFS、最小生成树、最短路径 |
| 查找 (SEARCH) | 5 | 查找 → 顺序查找、二分查找、哈希查找、树表查找 |
| 排序 (SORT) | 8 | 排序 → 冒泡、选择、插入、希尔、快排、归并、堆排序 |

**合计**：42 个节点，46 条关系

### 2.2 关系类型

| 关系类型 | 说明 | 示例 |
|----------|------|------|
| `PARENT_OF` | 层级包含关系（带 order 字段表示排序） | 线性表 -[PARENT_OF]-> 栈 |
| `RELATED_TO` | 知识关联关系（带 reason 字段说明关联原因） | 栈 -[RELATED_TO {reason: "递归遍历本质是栈的应用"}]-> 二叉树遍历 |

### 2.3 节点属性

每个 `KnowledgePoint` 节点包含以下属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| neo4j_id | String | 唯一标识，如 `KP_TREE_003` |
| name | String | 知识点名称 |
| code | String | 编号代码 |
| module | String | 所属模块（SET / LINEAR_LIST / TREE / GRAPH / SEARCH / SORT） |
| level | Integer | 层级深度（1=根模块，2=子模块，3=具体知识点） |
| difficulty | String | 难度等级（EASY / MEDIUM / HARD） |
| description | String | 知识点简介 |
| core_points | String | 核心考点 |

### 2.4 导入方式

数据通过 Docker 容器启动时自动导入（`neo4j-entrypoint.sh` → `auto_import.sh`），也可在 Neo4j Browser 中手动执行 `.cypher` 文件。

## 3. 后端

### 3.1 新增文件

| 文件 | 说明 |
|------|------|
| `src/main/java/com/learningplatform/controller/GraphController.java` | 图谱 REST 接口 |
| `src/main/java/com/learningplatform/service/GraphService.java` | 图谱业务逻辑，使用 Neo4jClient 执行 Cypher 查询 |

### 3.2 API 接口

| 方法 | 路径 | 说明 | 响应结构 |
|------|------|------|----------|
| GET | `/api/graph/data` | 获取完整图谱数据 | `{ nodes: [...], edges: [...] }` |
| GET | `/api/graph/node/{id}` | 获取单个节点详情及关联节点 | `{ id, name, code, module, level, difficulty, description, corePoints, relatedNodes: [...] }` |
| GET | `/api/graph/search?keyword=` | 按关键词模糊搜索节点 | `[ { id, name, code, module, ... } ]` |

### 3.3 前端 API 对接

**文件**：`frontend/src/api/modules/graph.js`（已有，无需修改）

```js
getGraphData()     // GET /graph/data
getNodeSummary(id) // GET /graph/node/:id
searchNodes(kw)    // GET /graph/search?keyword=
```

### 3.4 实现说明

- 使用 `Neo4jClient`（Spring Data Neo4j 提供）直接执行 Cypher 语句，不新增 `@Node` 实体类
- 复用已有的 `spring-boot-starter-data-neo4j` 依赖和 `application.yml` 中的 Neo4j 连接配置
- 返回结构与前端 API 层约定一致，统一通过 `Result<T>` 包装

## 4. 前端

### 4.2 修改文件

| 文件 | 说明 |
|------|------|
| `frontend/src/views/Graph/index.vue` | 从占位页面重构为完整的图谱可视化页面 |

### 4.3 GraphCanvas 组件

- **图表类型**：ECharts `graph`，力导向布局（force-directed）
- **节点样式**：
  - 颜色按模块区分（SET=灰色、LINEAR_LIST=蓝色、TREE=绿色、GRAPH=橙色、SEARCH=红色、SORT=紫色）
  - 大小按层级区分（level 1 = 50px，level 2 = 35px，level 3 = 25px）
- **边样式**：
  - `PARENT_OF`：实线 + 箭头
  - `RELATED_TO`：虚线 + 箭头 + 橙色
- **交互**：
  - 鼠标滚轮缩放、拖拽画布、拖拽节点
  - 悬停显示 tooltip（节点描述、难度、模块）
  - 点击节点触发 `node-click` 事件
  - 选中节点高亮关联节点和边（`focus: 'adjacency'`）
- **Props**：`nodes`、`edges`、`highlightIds`（搜索高亮）、`selectedId`（选中态）

### 4.4 NodeDetail 组件

- 显示节点名称、编号、难度标签、模块标签、层级
- 显示描述和核心考点（逗号分隔为标签展示）
- 显示关联知识点列表，点击可跳转
- 无选中节点时显示空状态提示

### 4.5 Graph 页面

- **顶部工具栏**：搜索输入框 + 模块筛选下拉 + 重置按钮 + 节点/边计数
- **主体布局**：左侧图谱画布 + 右侧详情面板（可收起）
- **搜索**：支持按名称、编号、描述模糊搜索，300ms 防抖
- **筛选**：按模块筛选节点，同步过滤关联的边
- **数据兜底**：API 请求失败时自动加载内置模拟数据（与 Neo4j 数据一致的 42 个节点）

### 4.6 页面路由

路由已存在（`/graph`），无需修改路由配置。

## 5. 模块依赖关系

```
知识图谱构建（数据层）
    ↓ 提供 Neo4j 节点/边数据
GraphService（后端）
    ↓ 提供 /graph/* API
Graph 页面（前端）
    ↓ 使用
GraphCanvas + NodeDetail（组件）
```

**上游依赖**：知识图谱构建模块需持续维护 `knowledge_points.cypher` 中的数据

**对外接口**：`/api/graph/data`、`/api/graph/node/{id}`、`/api/graph/search`
